# Система геймификации – Warmingtea

## 1. Концепция

Геймификация построена на модели Steam: профиль пользователя имеет уровень, который растёт по мере получения достижений. Достижения – единственный источник опыта. Никакого «фарма» повторяющихся действий: каждое достижение можно получить только один раз. Это гарантирует, что уровень профиля отражает реальный прогресс пользователя в платформе, а не количество потраченного времени.

---

## 2. Уровни профиля

| Параметр | Значение |
|---|---|
| Стартовый уровень | 0 |
| Стоимость одного уровня | **1 000 XP** |
| Максимальный уровень | ~40 (достигается при получении всех достижений) |
| Формула уровня | `floor(totalXP / 1000)` |
| Прогресс внутри уровня | `totalXP % 1000` из 1000 |

Уровень отображается в профиле в виде числа с прогресс-баром, аналогично Steam.

## 5. Баланс и кривая прогресса

### Первые сессии (0–2 часа использования)

Пользователь почти сразу получает 4–5 обычных достижений: создаёт задачу, выполняет её, добавляет тег. Это даёт ~2 500 XP и **уровень 2**, что немедленно вознаграждает новичка и даёт ощущение прогресса.

### Первая неделя

Выполнено ~50 задач, поддерживается 7-дневный стрик – это ещё ~4 000–5 000 XP. Пользователь достигает **уровня 5–7**. Каждый день есть шанс разблокировать что-то новое.

### Первый месяц

Активный пользователь достигает **уровня 10–15**. Редкие достижения начинают закрываться. Каждый новый уровень уже требует целенаправленного использования платформы.

### Долгосрочно (3–12 месяцев)

Эпические достижения требуют дисциплины (streak 30, 500 задач). Легендарные – это награда для преданных пользователей. **Уровень 30+** – статус «ветерана», который сигнализирует о серьёзном опыте.

### Почему не гринд

- Нет повторяемых источников XP – фармить нечего
- Достижения на количество задач распределены нелинейно (10 → 50 → 100 → 500 → 1000)
- Стрики мотивируют ежедневное использование, но сломать стрик – не катастрофа, так как XP уже получен

## 6. Отображение в профиле

### Блок уровня
```
[ Аватар ]  @username
            Уровень 7
            ████████░░░░░░  720 / 1000 XP
```

### Витрина достижений
- Закреплённые достижения (пользователь выбирает до 4 для показа на профиле)
- Вкладка «Все достижения» с фильтром по рангу и статусу (получено / нет)
- На заблокированных достижениях показывается иконка и название, но не условие (тайна мотивирует)

### Последние разблокированные
- Лента из 3 последних достижений с датой получения

## 7. Технические сущности

### AchievementDefinition (статичный список, не в БД)
```ts
interface AchievementDef {
  id:          string;        // 'first_task'
  title:       string;        // 'Первый шаг'
  description: string;        // 'Создать первую задачу'
  rank:        1 | 2 | 3 | 4; // 1=common, 4=legendary
  xp:          number;        // 500 | 1000 | 2000 | 4000
  icon:        string;        // lucide icon name или emoji
  secret:      boolean;       // скрыто до получения
}
```

### UserAchievement (в БД)
```
id          uuid PK
userId      uuid FK → users
defId       varchar  (ключ из статичного списка)
unlockedAt  timestamp
```

### Поля в User entity
```
xp          integer  default 0
level       integer  default 0   (или вычисляется: floor(xp/1000))
```

### AchievementsService
- `checkAndUnlock(userId, trigger)` – вызывается после каждого значимого действия (завершение задачи, создание тега, смена профиля и т.д.)
- `getUserProgress(userId)` – возвращает все достижения с флагом `unlocked`

## 8. Возможные улучшения

### 8.1 Витрина достижений на профиле
Пользователь закрепляет до 4 достижений, которые отображаются крупно с иконкой и рангом. Аналог значков в Steam.

### 8.2 Титулы (Titles)
За достижение определённых вех пользователь получает титул, который отображается рядом с именем:
- Уровень 5 → «Новичок»
- Уровень 10 → «Организатор»
- Уровень 20 → «Мастер»
- Уровень 30 → «Ветеран»
- streak_365 → «Железный»

### 8.3 Рамки профиля (Profile Frames)
Разблокируемые рамки/фоны для аватарки – визуальная награда без механического влияния. Мотивирует к персонализации.

### 8.4 Скрытые достижения
Часть достижений (`secret: true`) полностью скрыта – пользователь видит только «??? (Легендарное)». Неожиданный момент разблокировки особенно приятен.

### 8.5 Еженедельные испытания
Временные задания (сбрасываются в понедельник), за выполнение которых даётся небольшой фиксированный XP (например, 200 XP). Это единственный повторяемый источник XP – но только раз в неделю. Добавляет динамику без возможности фарма.

### 8.6 Уведомление о разблокировке
Toast-уведомление в правом нижнем углу при получении достижения:
```
🏆 Новое достижение!
«Неделя без остановки»  [Редкое]  +1 000 XP
```

### 8.7 Таблица лидеров (Leaderboard)
Опциональная публичная таблица по уровню профиля. Даёт социальный стимул для тех, кому важна соревновательность.

### 8.8 Достижения для будущих модулей
По мере добавления разделов платформы (Lists, Mail, Tunel) – добавляются новые достижения в соответствующих категориях. Пользователь видит «заглушки» для закрытых модулей, что стимулирует интерес к ним.

---

## 9. Принципы, которых стоит придерживаться

1. **XP только из достижений** – никакого ежедневного логина +10 XP. Уровень = реальный прогресс.
2. **Честная редкость** – легендарных достижений мало, и они действительно сложны. «Легендарный» уровень – это статус, а не просто цифра.
3. **Прозрачный прогресс** – на странице достижений видно, сколько осталось до следующего (например, «выполнено 73/100 задач»).
4. **Без наказания за пропуск** – платформа не штрафует за отсутствие. Стрик сбросится, но XP за прошлые дни уже получен.
